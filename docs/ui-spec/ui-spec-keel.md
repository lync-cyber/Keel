## 0. 设计方向

- **产品调性**: 稳固 · 克制 · 可信——像一份郑重的「应用宪章 / 海图」，而非花哨工具的廉价感。
- **目标用户场景**: 「重来者」（非程序员）在桌面端长时间使用；用过 Lovable/Bolt/v0 撞过「越改越乱」的腐化墙，看不懂代码，核心情绪诉求是确定感与掌控感。视觉要传达「结构稳固、看得懂、拿得走」。
- **视觉参考**: 海图 / 蓝图 / 郑重纸面文书；规避「彩虹式 AI slop」。核心隐喻 Keel = 龙骨，能力地图 = 应用的海图。
- **设计策略**: Calm Cartography（沉静海图 · 暖调蓝图）——暖纸底 `color.paper` + 航海藏青主色 `color.keel` + 三声部排版（衬线宪章标题 / 无衬线正文 / 等宽制图标签）；饱和色只留给语义（健康三态 + diff），其余一律暖中性。完整 token 见 ui-spec-keel-theme（THEME-01）。

## 1. 设计系统

设计系统 design token（色彩 / 排版 / 间距 / 圆角 / 阴影 / 动效 / 对比度与无障碍）的**完整单一事实源**见分卷 **ui-spec-keel-theme** §4 THEME-01，与 Penpot 文件「Keel」design tokens 一一对应。本主卷仅列关键 token 速览以便索引，数值以 theme 卷为准，组件与页面规格一律按 token 名引用。

### 1.1 色彩
关键 token 速览（完整色板 + 健康三态 soft 底 + diff 见 THEME-01 §4.1.1）：

| Token名 | 值 | 用途 | 设计意图 |
|---------|------|------|----------|
| color.paper | #F1ECE1 | 画布/页面底 | 暖纸，替冷灰给可信温度 |
| color.surface | #FAF7F0 | 面板/卡片底 | 比 paper 略亮，分层 |
| color.ink | #221F1A | 主文字/标题 | 暖墨近黑，对 paper 13.9:1 AAA |
| color.muted | #6F6657 | 次要文字/提示 | 暖 taupe，对 paper 4.8:1 AA 下限 |
| color.keel | #2B3F72 | 主色：选中/聚焦/主按钮/diff 新增 | 航海藏青，对 paper 8.7:1 |
| color.ok | #2F8F5B | 健康正常 | 绿（情绪信号层） |
| color.warn | #C8841A | 健康小问题 | 暖琥珀 |
| color.alert | #CC4636 | 健康要处理 | 暖朱 |

**对比度验证**: 正文文字与背景对比度 ≥4.5:1（ink 13.9 AAA、keel 8.7、muted 4.8 AA 下限）；健康色仅用于 pill/图标/大字与各自 soft 底配对，逐对核验见 THEME-01 §4.1.4。

### 1.2 排版与间距
三声部排版（font.serif/sans/mono）、字号阶梯、间距/圆角/阴影/动效 token 见 THEME-01 §4.1.2–§4.1.3，本卷不复述。

组件清单（UC-001..UC-020）见分卷 **ui-spec-keel-components** §2。组件按 token 绑定主题，所有「界面内容」字段一律取蓝图 `plain_*`，技术字段不上界面。

## 3. 页面布局

页面规格分两卷：

- **ui-spec-keel-pages-core** §3（P-001..P-006）：核心屏——三面工作区外壳、能力地图、意图对话+蓝图 diff 确认、应用实时预览、实现进度、健康调和。覆盖 F-002~F-011。
- **ui-spec-keel-pages-lifecycle** §3（P-007..P-011）：生命周期屏——新项目上手+引擎安装、意图日志时光机、一键上线+安全体检、导出交接、自由实现区。覆盖 F-012~F-018。

## 4. 导航与路由

Keel 工作区前端为 React + Vite Web 壳（arch §1.4），形态接近桌面应用的单工作区 SPA：核心是 P-001 三面工作区，生命周期能力以**抽屉 / 模态**叠加于工作区之上，而非跳转独立全页路由——非程序员心智模型是「我的应用始终在眼前，操作是对它做某件事」。

| 路由 | 页面 | 承载方式 | 权限 |
|------|------|---------|------|
| `/` | 项目选择 / 新建（无项目时重定向 onboarding） | 全页 | 已登录 |
| `/onboarding` | 引擎安装+登录+描述意图+生成骨架（P-007） | 全页向导 | 公开（登录前可达安装步） |
| `/project/:id` | 三面工作区外壳（P-001，默认屏） | 全页 | 项目所有者 |
| `↳ 对话面 / 地图面 / 预览面`（P-002/P-003/P-004） | 工作区内三面 | 同屏面板（非路由） | 同上 |
| `?panel=timeline` | 意图日志时光机（P-008） | 右侧抽屉 | 同上 |
| `?panel=health` | 健康调和（P-006） | 右侧抽屉 / 地图叠加 | 同上 |
| `?dialog=diff` | 蓝图 diff 确认（P-003） | 居中模态 | 同上 |
| `?dialog=decision` | 需你决策卡（P-005，UC-010） | 居中模态（阻断态） | 同上 |
| `?dialog=deploy` | 一键上线+安全体检（P-009） | 居中模态 | 同上 |
| `?dialog=export` | 导出交接（P-010） | 居中模态 | 同上 |

- **唯一变更入口纪律**: 任何结构变更只能经对话面意图捕获（F-009 AC-007）→ diff 确认模态；地图只读、无直接编辑路由（F-006 AC-006）。
- **URL query 状态**: 抽屉 / 模态以 query 参数表达，保证可深链与浏览器后退关闭，不引入独立页面跳转打断「应用在眼前」的连续感。

## 5. 响应式策略

v1 主战场为桌面长时使用；以三面工作区的折叠行为为响应式核心。

| 断点 | 宽度 | 布局变化 |
|------|------|---------|
| comfortable | ≥1440px | 三面并列：对话面（固定 ~360px，左）· 能力地图面（弹性主区，中）· 应用预览面（~420px，右）。顶栏健康灯 + 项目名常驻 |
| compact | 1024–1439px | 三面并列但收窄；预览面可折叠为右侧标签（点击展开覆盖中区），地图始终可见；对话面可收为图标轨 |
| narrow | 768–1023px | 单活动面 + 顶部 segmented control 切换 对话/地图/预览（M-015「只用对话面即可完成构建」承诺在此断点尤为关键）；抽屉占满宽 |
| mobile | <768px | 非 v1 目标（桌面构建工具）；降级为只读能力地图 + 对话查看，提示「在桌面端继续构建」，不提供完整三面交互 |

- **首屏性能约束**: 三面初始挂载（对话 + 20 节点地图 + 预览首挂）≤2s（F-008 AC-005 / arch §5.1）；能力地图渲染面板代码分割懒加载，避免地图库 bundle 拖慢对话面首屏（选型权衡见 ui-spec-keel-pages-core P-002）。
- **可达性基线**: 三面均键盘可达；地图节点 Tab 序按依赖拓扑；健康色不单独承载语义（伴随图标 + mono 文案），对比度门槛见 THEME-01 §4.1.4。
