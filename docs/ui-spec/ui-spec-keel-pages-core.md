## 3. 页面布局

### P-001: 三面工作区外壳（Workspace Shell）
- **路由**: `/project/:id`（默认主屏）
- **使用组件**: UC-001（顶栏健康灯）, UC-002, UC-006, UC-018, UC-020
- **空间构成**: 顶栏（~56px，左 wordmark+项目名 `font.serif`，中留白，右 UC-001 总健康灯 + UC-020 用量）；主区三面并列——对话面（左固定 ~360px）· 能力地图面（中弹性主区，视觉重心）· 应用预览面（右 ~420px）。暖纸 `color.paper` 底，面板 `color.surface` + `radius.lg` + 极淡蓝图网格纹理仅地图面。
- **布局描述**: 三面为同屏面板而非路由（P-002/P-003/P-004 分述面内容）；面间 1px `color.line` 分隔可拖拽调宽。用户可完全只用对话面完成构建（地图/预览为增强理解，非操作前提，F-008 AC-004）。外壳保留 ShellAdapter 接缝（Web↔桌面，M-015）。
- **状态流**:
  - loading: 三面骨架屏，目标 ≤2s 完成首次挂载（F-008 AC-005）；地图面板代码分割懒加载。
  - empty: 新项目未生成骨架时，地图/预览面显 UC-017，对话面引导「描述你想要的应用」。
  - populated: 三面常驻，对话驱动、地图反映结构、预览反映运行态。
  - error: 单面失败降级为该面 UC-017 通俗态，不整屏崩；绝不出现技术错误页。
- **映射功能**: F-008（M-015）

### P-002: 能力地图面（Capability Map）
- **路由**: `/project/:id` 内中区面板
- **使用组件**: UC-002（节点）, UC-003（连线）, UC-004（原语徽章/图例）, UC-005（详情卡）, UC-001（健康叠加）
- **空间构成**: 弹性主区铺满中面；极淡蓝图网格背景；节点为暖投影实体卡片错峰落位（staggered，像海图被绘出）；左下角 UC-004 七类图例 + 健康三态图例；右上层级聚合/焦点子图切换。
- **布局描述**: 100% 从蓝图 `plain_*` 渲染（不从代码逆推，F-006 AC-001）；七原语全量译为业务语言（UC-004）；健康叠加读 E-005 HealthSnapshot（绿/黄/红，红色可点进 UC-009）；节点 status 来自 M-011/M-006 进度（UC-013）；大图层级聚合（折叠子图）+ 任务焦点子图（只展示当前变更相关节点，F-006 AC-007）；**只读**——结构变更一律走对话面意图流程（F-006 AC-006），地图上无编辑入口。数据契约见 API-011。
- **渲染选型决策**:
  - **推荐**: React Flow v12（`@xyflow/react`）。理由——内建 节点/边/自定义节点渲染、pan/zoom、子流程折叠（sub-flows）、minimap，直接对应 F-006 的层级聚合 + 焦点子图 + 健康叠加 + 只读投影需求；自定义节点便于绑定 THEME-01 token 还原 UC-002 实体卡片；只读经 `nodesDraggable=false / nodesConnectable=false / elementsSelectable=true` 平凡实现；20 节点规模（F-006 AC-008 渲染 ≤1s）下性能非瓶颈。
  - **唯一风险**: bundle 体积对三面首屏 ≤2s（F-008 AC-005）的影响。**缓解**: 地图面板 `React.lazy` 代码分割懒加载，使对话面首屏不被地图库阻塞；核心 `@xyflow/react` 体积在 React 19 + Vite 应用中占比小。
  - **备选**: D3 自绘（`d3-force`/`d3-zoom`/`d3-selection`）——体积更小但需手写布局/命中测试/React 协调，开发成本显著高。
  - **重新评估条件**: dev_planning/development 阶段对 React Flow 实际 bundle 做 profiling，若懒加载后三面首屏仍 >2s，则回退 D3 自绘并仅引入所需 d3 子模块。本决策标 [ASSUMPTION]，最终 binding 在开发阶段经实测确认（arch §1.4 已点名 React Flow 候选）。
- **状态流**:
  - loading: 节点错峰落位动画；聚合层级默认极简、按需展开（F-006 AC-007）。
  - empty: UC-017 map-empty「还没有能力，先描述一个吧」。
  - populated: 节点+连线+健康叠加；点击节点→UC-005 详情卡。
  - error: HealthSnapshot 不可达时节点显 health 未知灰态 + 通俗提示，不阻塞地图浏览。
- **映射功能**: F-006（M-009）, F-007 高亮联动（UC-003 blast-highlight）

### P-003: 意图对话 + 蓝图 diff 确认（Intent → Blueprint Diff）
- **路由**: 对话面（`/project/:id` 左面）+ `?dialog=diff` 居中模态
- **使用组件**: UC-006（对话气泡）, UC-007（diff 卡）, UC-008（影响预览条）, UC-018
- **空间构成**: 对话面竖向消息流 + 底部输入框（支持「指图说话」：先选地图节点再输入，F-009 AC-003）；diff 确认为居中模态覆盖工作区，模态内顶 UC-008 影响预览、中 UC-007 三组 diff、底三按钮。
- **布局描述**: 自然语言输入 → 产出蓝图 diff（API-005）→ 以「地图 diff + 通俗解释」呈现（人话变更说明书，F-009 AC-002），绝无代码 diff；影响预告复用 F-007 blast radius 并与地图高亮联动；确认 → 契约冻结门（F-009 AC-006，UC-007 转 frozen）；追加 → 回对话生成新 diff；拒绝 → 丢弃。任何结构变更必经此流程（F-009 AC-007），不存在绕过蓝图直接写代码的路径。
- **状态流**:
  - loading: diff 生成中（目标 ≤10s，F-009 AC-008）对话显 UC-006 working「正在理解你的意图…」。
  - empty: 对话面初始引导文案 + 起步示例。
  - populated: diff 卡呈现 added/modified/touched + blast radius；可确认/追加/拒绝。
  - error: 契约不完整/不一致 → 冻结门退回澄清（UC-006 system-clarify），通俗追问而非报错。
- **映射功能**: F-009, F-007（M-005, M-009）

### P-004: 应用实时预览面（App Preview）
- **路由**: 预览面（`/project/:id` 右面）
- **使用组件**: UC-017（降级空态）, UC-001, UC-015（刷新 Toast）
- **空间构成**: 右面嵌入目标应用真实运行界面（iframe/代理 dev server），可直接交互操作（F-008 AC-001）；顶部细栏显当前预览状态（ready/building/unavailable）。
- **布局描述**: 门禁与回归守护通过后自动刷新到最新（无需手动，F-008 AC-002），刷新完成弹 UC-015 success Toast；预览不可用时（骨架未生成/构建中）显 UC-017 通俗状态说明，不出现技术错误页（F-008 AC-003）。数据契约见 API-012。
- **状态流**:
  - loading: building 态 UC-017「应用正在搭建，稍候即可预览」。
  - empty: unavailable 态（骨架未生成）引导回对话面发起首个意图。
  - populated: ready 态嵌入真实运行界面，可交互；自动刷新。
  - error: 预览代理失败降级为 unavailable 通俗态，不暴露构建日志。
- **映射功能**: F-008（M-010）

### P-005: 实现进度（Implementation Progress）
- **路由**: `/project/:id` 地图叠加 + `?dialog=decision` 阻断模态
- **使用组件**: UC-013（状态标）, UC-014（修复中条）, UC-010（需你决策卡）, UC-002
- **空间构成**: 进度以能力地图节点状态变化呈现（规划中→实现中→已就绪，F-010 AC-005），无独立进度页；自愈时对话/节点显 UC-014；冲突时 UC-010 居中阻断模态。
- **布局描述**: 契约冻结通过后进入实现（F-010 AC-001）；按依赖拓扑序逐切片实现，被依赖切片先就绪，用户在地图观察节点依序进入 ready（F-010 AC-002）；每次改动经验证门（F-003），硬违规静默自愈（UC-014），软问题记健康；用户全程不看代码或终端（F-010 AC-007）。门禁反复硬阻断且自愈无法推进 → UC-010 需你决策卡（F-010 AC-006）。实现隔离：失败/中止时应用回到改动前完整可用状态，已就绪切片不受影响（F-010 AC-004）。
- **状态流**:
  - loading: 节点 implementing 态 + UC-014「正在实现…/正在修复…」。
  - empty: 无进行中实现时不显进度叠加。
  - populated: 节点依序 ready；预览面（P-004）随之自动刷新。
  - error: 自愈达上限（F-005 AC-004）→ UC-010 升级；中止 → 回滚到改动前态。
- **映射功能**: F-010, F-005, F-004（M-006, M-007）

### P-006: 健康调和（Health & Reconcile）
- **路由**: `?panel=health` 右侧抽屉 + 地图健康叠加
- **使用组件**: UC-009（健康问题卡）, UC-001, UC-003（diff 态连线）, UC-016（重做确认）
- **空间构成**: 健康抽屉列出问题清单（UC-009 条目）；地图同步红/黄节点叠加（P-002）；每项含通俗标题 + 一键修复。
- **布局描述**: 漂移以通俗语言展示（「收藏功能被实现了两遍，建议合并」，F-002 AC-003），不暴露代码术语；问题分结构违规/回归破坏/软告警（对应 API-002）；点选某项 → 通俗说明 + 一键修复（F-011 AC-006）→ 调和引擎以受影响切片为单位整片重生成（F-011 AC-001），受验证门约束（F-011 AC-003），调和后该漂移消失且不引入回归（F-011 AC-004）。整片重生成前若切片含 escape hatch 或未入图内容 → 先弹 UC-016「这块有你的自由实现区/未入图内容，重做可能丢失」确认（F-011 AC-007）。双向裁决：漂移可「回收代码服从蓝图」（调和）或「更新蓝图承认实际」（回到 P-003 diff 确认，F-002 AC-004）。
- **状态流**:
  - loading: 体检/检测中（增量检测 ≤3s，F-002 AC-005）。
  - empty: 全绿态「应用结构健康」无问题清单。
  - populated: 问题清单 + 一键修复；修复后条目消失、节点转绿。
  - error: 调和反复失败无法自愈 → UC-010 升级（F-011 备注），不静默放弃。
- **映射功能**: F-002, F-003, F-004, F-005, F-011（M-002, M-003, M-004, M-007）
