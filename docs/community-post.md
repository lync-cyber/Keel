# 我想做个东西，让不会写代码的人造的 App 不要越改越烂——求骂醒

先自我介绍一下我的"成就"：我用 Lovable / Bolt 这类 AI 工具做过好几个小项目，最擅长的事，是把一个本来能跑的东西，靠"再帮我改一下"改到它亲妈都不认。

流程大概是这样的：第一天，神了，三句话一个能登录能发帖的网站就出来了，感觉自己马上要财务自由。第二个礼拜，加个收藏功能，AI 顺手把登录改崩了。我让它修登录，它把收藏又干掉了。到第三个礼拜，代码我是一行都看不懂的，AI 也开始一本正经地胡说八道，最后我对着一坨没人能读懂的东西，做了每个非程序员都会做的终极操作——**推倒重来**。

这个循环我跑了不止一次。后来我开始怀疑：是不是只有我这么菜？翻了一圈发现，不是。**"起步飞快、长大就烂"几乎是这类工具的通病。** 你能很快做出来，但你没法让它持续长大。

## 那为什么程序员的项目不会这样？

会，但他们有办法兜住。开发者圈子里早就有一堆"防腐"的实践：spec-driven development（先写规格再写代码）、架构门禁（CI 里卡住不合规的改动）、依赖检查工具……这些东西的作用，就是不让项目在一次次改动里慢慢变形。

问题是——**这些全是给会读代码的人用的。** 让我去写 spec、看 diff、配 dependency-cruiser？那我还不如直接去学编程。

所以我钻进去的牛角尖是：**能不能把程序员这套"结构纪律"，做成不会写代码的人也能用的样子？**

## Keel 的想法

我给它起名叫 **Keel（龙骨）**——就是船底那根承重的大梁，船在风浪里不翻，全靠它。寓意有点中二，但我喜欢。

核心就两句话：

**一、应用蓝图（Blueprint）是唯一的真相源。** 不是代码说了算，是这张蓝图说了算。蓝图是一份声明式、能版本化、机器能读能检查的"应用应该长什么样"。代码必须永远服从蓝图，而不是反过来。

**二、确定性门禁防止结构漂移。** 每次改动都要过一道不能绕开的检查：你这个功能是不是偷偷伸手进了另一个功能的内部?同一件事(比如"判断登录没登录")是不是被实现了两遍?违反了就直接拦下来,自动重试。

而你——作为那个不会写代码的人——**不看代码,你看一张"能力地图"。** 你的 App 有哪些能力、它们怎么连在一起、加这个新功能会动到哪几块,全在图上用人话标出来("会碰这 3 个区块,不会碰登录")。你在图上理解、审查、拍板,代码的事交给 AI 在约束里去填。

哦对,还有一条我自己很在意的:**代码全程归你,随时能一键导出迁走。** 导出的就是标准的 JS/TS 仓库,带着那些检查工具的配置,离开 Keel 也能自己管住自己,不绑架你。

## 我得诚实交代的边界

这部分我特意放出来,因为我最怕做成一个吹得天花乱坠的 PPT。

Keel **只机械化拦截"结构性"的错误**——接口、依赖、分层、契约这些。它管不了"逻辑对不对"。"逻辑对不对"靠图驱动生成的测试去兜,我不会吹它能保证你的业务逻辑零 bug。还有像"这个交互体验好不好"这种没法写成机器判据的事,老老实实回退到人工确认。

换句话说,它的承诺很窄:**不保证一次写对,但保证不会越长越乱。** 我宁可范围小但兑现,也不想又做一个"看着啥都行、用起来啥都不行"的东西。

## 跟现有的东西比

- 对 Lovable / Bolt / v0 / Replit:它们赢在起步,我赌的是成长期不腐化。
- 对 Floot / Mocha 这种:它们把你锁在平台里;Keel 开放代码、不锁定、能迁出。
- 对 Spec Kit / Kiro 这些:它们要求你会写 spec、读 diff;Keel 想把同样的纪律做成非程序员能用。

## 然后是我最没底的地方(真心求拍)

整个东西成不成立,压在**一个我还没验证的假设**上:

> **一份蓝图,能不能同时做到"机器能检查"和"非程序员能看懂"?**

这两件事天然打架。机器要的是精确、无歧义、结构化;人要的是说人话、别让我看符号。我赌它们能在同一份东西里共存(界面只渲染"人话字段",技术细节压根不上界面),但**这只是我的赌注,不是事实。** 如果这个假设是错的,那 Keel 就是个很美的、会在第一关就死掉的点子。

目前我手上有三张静态的 Web 原型(能力地图长什么样、加功能怎么交互),正准备拿去做可用性测试,就想知道:一个没写过代码的人,看着那张图,能不能答上来"我这 App 有哪些能力、它们怎么连的"。这一关过不了,后面再花哨都白搭。

所以想听听大家的:

1. "起步快、长大烂"这个痛,你也踩过吗,还是只有我这么倒霉?
2. "蓝图既能机器检查、又能让小白看懂"——你觉得这事儿现实吗?哪里最容易崩?
3. 有没有我没注意到的、已经在做类似事情的项目?(被指出"这早有人做了"我也认,总比闷头造三年好。)

骂得越狠越好,我谢谢你。
