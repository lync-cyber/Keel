# 0011. 聚焦非程序员单一产品，开发者工具产品线移出 PRD 范围

状态：accepted（2026-06-12）

## 背景

PRD v1.x 以「一核两壳」产品矩阵表达：共享内核 + 壳 A（开发者版：arch map / CI/PR 架构检查 / MCP/CLI 接入）+ 壳 B（小白版：非程序员 Web 产品）。两个面向不同人群的产品形态混在同一份功能清单（F-015/F-016 为壳级「专属能力包」），导致产品定义模糊、功能粒度失衡，且与既定项目定位（让非程序员零代码构建不腐化应用）存在张力。

## 决定

Keel 定义为**面向非程序员的单一产品**。开发者只保留「接手者」间接角色（导出物与交接包的验收对象）。开发者工具产品线（存量仓库 arch map、CI/PR 架构检查、面向开发者用户的 MCP/CLI/IDE 插件产品入口、图级 PR approve）整体移出本 PRD 范围；如未来启动，须另立独立 PRD，不与本产品功能清单混编。

## 后果

- PRD v2.0 重写：删除「一核两壳」「内核/壳 A/壳 B」概念与术语；功能清单按能力域统一编号（F-001..F-018），壳级「专属能力包」拆解为独立功能。
- 原 F-015（开发者版三档）、流程 E（开发者版获客钩子）、arch map 性能指标随之移除。
- 重新评估条件：R1 核心假设验证通过且主产品进入稳定迭代后，可重新评估开发者产品线立项。
- 边界澄清：移出的是「人类开发者作为最终用户」的产品形态；执行引擎（LLM）消费 Keel 门禁/蓝图/地图能力的**内部工具接缝**（ACP / MCP Server / hooks / CLI）属产品内部机制，保留在范围内（见 PRD F-018 AC-006）。
