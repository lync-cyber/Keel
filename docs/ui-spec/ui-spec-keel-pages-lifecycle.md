## 3. 页面布局

### P-007: 新项目上手 + 引擎安装/登录（Onboarding）
- **路由**: `/onboarding`（无项目时由 `/` 重定向）
- **使用组件**: UC-019（引擎安装/登录卡）, UC-020（用量）, UC-006（对话）, UC-017, UC-018
- **空间构成**: 全页分步向导（`font.serif` 大标题 + 步进指示），暖纸底，单列居中卡片流；步骤：① 检测引擎 → ② 图形安装引导（如需）→ ③ 订阅登录/API Key → ④ 描述应用意图或选模板 → ⑤ 生成初始蓝图与骨架 → 进入工作区。
- **布局描述**: 检测本地执行引擎就绪性（Claude Code/Codex 桌面应用/CLI/ACP adapter，F-012 AC-001），未装则 UC-019 图形安装引导（绝无终端命令）；鉴权优先 OAuth 订阅登录、兼容 API Key、本地存储最小权限（F-012 AC-002）；后台经 ACP/SDK/CLI 连接并冒烟校验连通（F-012 AC-003，桌面图形界面仅装+登录不驱动）；用户描述意图（自由输入或选模板，UC-006）→ 自动选定 `modular-monolith` 原型 → 生成初始蓝图与最小可运行骨架（F-012 AC-004），初始蓝图在能力地图完整渲染（所有原语含 `plain`，F-012 AC-005）；顶栏 UC-020 显引擎侧订阅/用量（F-012 AC-007）。从安装到可运行骨架，全程未见代码或终端（F-012 AC-006）。
- **状态流**:
  - loading: 引擎检测中 / 冒烟校验中 UC-019 detect 态。
  - empty: 未装引擎 → UC-019 install-guide 图形步骤。
  - populated: 登录成功 + 冒烟通过 → 描述意图 → 骨架生成 → 跳 P-001。
  - error: 安装/连接失败显通俗重试引导，不暴露安装日志/错误码。
- **映射功能**: F-012（M-014）, F-018 用户可见接入（M-008）

### P-008: 意图日志时光机（Intent Log & Time Machine）
- **路由**: `?panel=timeline` 右侧抽屉
- **使用组件**: UC-012（时间线条目）, UC-016（回滚确认）, UC-018
- **空间构成**: 右抽屉竖向时间线，自上而下倒序；每条 UC-012（自然语言摘要 + 时间），current 点 `color.keel` 高亮；条目类型区分变更/决策/上线。
- **布局描述**: 每次变更自动记录意图日志（原始意图文本、蓝图 diff、时间戳；需你决策留痕同样记，F-013 AC-001）；时间线以自然语言描述每次变更（「添加了用户收藏文章功能」，F-013 AC-002），不暴露提交哈希/git 命令/代码 diff；任一历史点可查其意图与蓝图 diff（查询 ≤3s，F-013 AC-003）；时光机「回到这里」一键回滚到任意点，蓝图+代码+健康灯同步恢复（F-013 AC-004），回滚后为一致状态（F-013 AC-005）。回滚前若检测到时间线未记录的通道外改动 → 先弹 UC-016「有没记录在时间线上的改动，回到过去会丢掉它们」确认（F-013 AC-006），不静默丢弃。
- **状态流**:
  - loading: 时间线加载骨架。
  - empty: 仅初始化条目时显「应用的故事从这里开始」。
  - populated: 完整时间线 + 回滚入口。
  - error: 回滚遇通道外改动 → UC-016 untracked-rollback 确认（阻断式）。
- **映射功能**: F-013（M-011）

### P-009: 一键上线 + 安全体检（Deploy & Security Audit）
- **路由**: `?dialog=deploy` 居中模态
- **使用组件**: UC-011（安全红绿灯卡）, UC-016, UC-018
- **空间构成**: 居中模态，顶安全分（`font.mono` 大字）+ 三类红绿灯逐项（认证配置/数据权限/密钥泄露，UC-011）；底「上线」主按钮（has-red 时 disabled）。
- **布局描述**: 一键上线入口触发后自动构建并部署到锁定托管环境，完成返回可访问地址（UC-015 success Toast，全程无终端/部署配置，F-014 AC-001）；部署前安全体检三类检查 + 安全分逐项红绿灯（F-014 AC-002）；存在红灯阻断上线，逐项通俗说明 + 修复入口（可触发调和），全绿才部署（F-014 AC-003）；上线动作记入意图日志、可回滚到上线前（F-014 AC-004，关联 P-008）。
- **状态流**:
  - loading: scanning 态体检中 / 部署中进度。
  - empty: 不适用（入口即触发体检）。
  - populated: all-green → 可上线；has-red → 阻断 + 修复入口。
  - error: 部署失败通俗提示 + 重试，不暴露部署日志；时间线保留上线前可回滚点。
- **映射功能**: F-014（M-012）

### P-010: 导出交接（Export & Handoff）
- **路由**: `?dialog=export` 居中模态
- **使用组件**: UC-016, UC-018, UC-017
- **空间构成**: 居中模态两区——① 一键导出标准仓库；② 一键生成开发者交接包；各附说明文案。
- **布局描述**: 随时导出结构服从蓝图的标准 JS/TS 仓库（F-015 AC-001），含编译落地的标准工具配置（dependency-cruiser/eslint/tsconfig/JSON Schema），脱离 Keel 仍可用标准工具自我强制结构（F-015 AC-002）；导出物可独立运行、无 Keel 专有运行时依赖、无凭据/内部地址硬编码（F-015 AC-003/AC-004）；导出完成提示「此项目已包含架构约束工具配置，脱离 Keel 亦可自我强制」（F-015 AC-005）。交接包一键生成（蓝图→架构文档 + 意图日志→决策史 + 边界规则→上手指南，Markdown 独立可读，可含技术视图投影，F-016 AC-001/AC-002/AC-003）。强调「代码全程归你、可随时带走」的产品承诺。
- **状态流**:
  - loading: 导出/打包生成中进度。
  - empty: 不适用。
  - populated: 导出完成 → 下载入口 + 自我强制说明；交接包完成 → 下载入口。
  - error: 生成失败通俗提示 + 重试。
- **映射功能**: F-015, F-016（M-013）

### P-011: 自由实现区（Escape Hatch）
- **路由**: `/project/:id` 地图节点 + `?dialog=diff` 标注流程
- **使用组件**: UC-004（自由实现区徽章）, UC-005（详情卡含意图）, UC-007（diff 标注）
- **空间构成**: escape hatch 模块在能力地图以「自由实现区」显著标识（UC-004 `color.brass` 描边 + 虚线节点）区分普通模块，并展示附加意图说明（UC-005 escape-hatch 变体）。
- **布局描述**: 经意图流程（P-003）将任意功能区块标注为自由实现区，必填自然语言意图说明（intent 为空则拒绝，F-017 AC-001）；地图上显著标识 + 展示意图 + 提示「该区域不受图约束保护」（F-017 AC-003）；标注不传播——依赖方仍受完整图约束（F-017 AC-004），视觉上仅被标注节点本身带自由区样式；调和触及自由实现区时先弹 UC-016 确认「重做可能丢失」（关联 P-006 / F-011 AC-007），可选「仅提示不重生成」。
- **状态流**:
  - loading: 标注 diff 生成中（复用 P-003 流程）。
  - empty: 无自由实现区时地图无此类标识。
  - populated: 自由实现区节点带 brass 标识 + 意图说明可查。
  - error: intent 为空 → 通俗校验拒绝标注（UC-006 system-clarify 追问意图）。
- **映射功能**: F-017（M-001, M-009）
